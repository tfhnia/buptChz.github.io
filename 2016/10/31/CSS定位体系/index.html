    CSS盒子模型

    CSS盒子详解

    width/height用于设置内容（content）的宽高；
    padding用于控制元素的内边距，也就是设置内容和边框间距；
    margin用于控制元素的外边距，即控制该元素与其他元素之间的距离；
    所以，元素内容（content）与其他元素的距离其实是同时取决于padding和margin的属性。
    关于margin，值得一提的是，margin可以是负值，margin可以取负，值为padding的相反数时，元素表现为内容的边界扩展到元素的边界。
    并且，你可能也会遇到margin合并问题，即当两个垂直外边距相遇时，它们将形成一个外边距，这时候的外边距取决于较大值。

    注意，padding/border/margin可以设置上右底左四个方位。内边矩（padding-top、padding-right、padding-bottom、padding-left）、边框（border-top、border-right、border-bottom、border-left）和外边距（marging-top、margin-right、margin-bottom、margin-left）。

    关于margin和width的auto取值
    1. 正常流中的块级元素框的水平总和就等于父元素的width
    2. 在水平格式化的“7大属性”中，只有3个可以设置为auto: width, margin-left, margin-right;

    在CSS2的标准里Visual formatting model details，指出当判定容器的盒子结构时，这里有5条if判断规则。
    如果 width 是 auto ，这时当 ml/mr 是 auto，则margin-left和margin-right就是0，后面的一条 if 不生效了。
    如果 width 不是 auto，即width有确定宽度了。当 ml/mr 是 auto，浏览器计算出 ml+mr = 父容器宽度 - width宽度，且ml == mr，这时就居中。

    设置width和height所指定的盒子

    box-sizing 可以指定为border-box，这时，指定一个元素的width和height即指定了border+padding+content的宽高。
    box-sizing 是个很新的属性，目前你还应该像我之前在例子中那样使用 -webkit- 和 -moz- 前缀。这可以启用特定浏览器实验中的特性。同时记住它是支持IE8+。
