Git的一些常用指令

场景设置：

  1. 现在有个master分支，由项目负责人创建；另有dev分支，用于发布每一版的项目
  2. 项目人员分配
    1. 郭同学 — 负责后台与前端交接 => 创建有分支 gdh
    2. 袁同学 — 负责后台管理 => 创建有分支 yht
    3. 陈同学(鄙人) — 负责前端报名表单 => 尚未创建分支

现问题：

  1. 新建仓库并克隆项目至本地
  2. 查看所有的分支
  3. 拉取dev分支到本地 git
  4. 拉取gdh分支到本地
  5. 修改gdh分支中的前端页面，pull request请求合并
  6. 创建自己的分支chz
  7. 拉取gdh分支至chz分支
  8. 删除本地gdh分支
  9. pull本地chz分支
  10. 比对chz和gdh两个分支的不同
  11. 分支合并

本地的chz仓库提交至gdh仓库。gdh仓库已经修改。现拉取最新的gdh仓库至chz仓库：

  1. git checkout master {跳转到其它分支}
  2. git branch -D chz {删除本地chz分支}
  3. git fetch origin gdh:chz {拉取远程gdh仓库至chz仓库}
  4. git checkout chz {跳转至chz仓库}

其它问题：

  1. github的fork和clone有什么区别？
    fork：在github页面，点击fork按钮。将别人的仓库复制一份到自己的仓库。
    clone：将github中的仓库克隆到自己本地电脑中
  2. 将本地的仓库强制推到远程（无视远程版本冲突）
    git push –set-upstream origin +chz
